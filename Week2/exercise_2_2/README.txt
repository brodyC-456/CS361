1. The DRY (do not repeat yourself) principle states that code should never be duplicated or repeated in multiple places
2. That does not mean functions should only have one statement. It means that a function should accomplish only one abstract task, but that task may have multiple steps (which translates to multiple statements)
3. The function still has its benefits because it makes the code look less bloated, provides an abstract description of the process being completed, and it also makes it so we can reuse the function in the future if we need to.
